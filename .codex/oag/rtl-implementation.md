# OAG RTL Implementation

RTL may be generated by an LLM. That makes behavior/cycle contracts more
important, not less important.

## Core Rule

```text
LLM generates RTL.
RTL is not truth.
RTL implements the locked contract.
TB independently predicts from the same contract.
Validator checks that RTL and TB are tied to the same contract oracle.
```

The RTL generator may use full engineering judgment for implementation shape.
It may not invent design meaning.

## RTL Agent Posture

```text
You are free to choose implementation structure.
You are not free to change design truth.
RTL is an implementation of the locked contract.
RTL may optimize, organize, name, and structure internals.
RTL may not invent behavior, timing, reset values, address map, priorities, or protocol semantics.
```

Use `.codex/oag/rtl-dialect-policy.md` for the default RTL dialect,
`.codex/oag/rtl-ppa-principles.md` for PPA-aware structure choices, and
`.codex/oag/domain-crossing-principles.md` for CDC/RDC domain-safety choices.

## Allowed RTL Freedom

The RTL agent may choose:

- internal signal names;
- edge-triggered or combinational `always` block organization within the RTL
  dialect policy;
- address decode implementation style;
- register update code style;
- helper functions when accepted by the RTL subset;
- `case` versus `if` structure;
- masking implementation;
- internal file organization within the assigned scope.

These choices are implementation details. They are acceptable when the external
contract is preserved.

## Forbidden RTL Freedom

The RTL agent must not change:

- port meaning;
- APB timing;
- reset value;
- reset polarity;
- address map;
- mapped/unmapped access policy;
- `PSLVERR` policy;
- IRQ W1C priority;
- DATA_IN synchronization stage count;
- clock/reset domain classification;
- CDC/RDC mitigation pattern;
- `PREADY` policy;
- read-only write behavior;
- side effects not present in the contract.

If the behavior/cycle rule is missing, the RTL agent must stop with a precise
blocker instead of inventing it.

## Shared Truth Flow

RTL and TB consume the same contract oracle but do different jobs:

```text
locked requirement
  -> obligation
    -> contract
      -> behavior_model / cycle_rules
        -> RTL implementation
        -> TB expected oracle
        -> scoreboard row
        -> validation
```

RTL agent:

```text
Implement behavior_model and cycle_rules.
```

TB agent:

```text
Compute expected values from behavior_model and cycle_rules.
```

Evidence validator:

```text
Check that TB followed the contract, not the RTL.
Check that RTL claims trace to the same contract oracle.
```

## RTL Generation Input

An RTL authoring packet should give the LLM enough truth to avoid guessing:

```yaml
module:
  name: apb_gpio
  parameters:
    GPIO_WIDTH: 32
interface:
  bus: APB
  clock: PCLK
  reset: PRESETn
  reset_active: low
registers:
  DATA_OUT:
    address: 0x00
    reset: 0
    access: rw
  DATA_IN:
    address: 0x08
    access: ro
    source: gpio_in_sync2
behavior_model:
  DATA_OUT.write:
    effect: data_out_next = PWDATA[GPIO_WIDTH-1:0]
  DATA_OUT.read:
    effect: PRDATA = data_out
cycle_rules:
  apb.write_sample:
    when: PSEL && PENABLE && PWRITE
  apb.read_response:
    when: PSEL && PENABLE && !PWRITE
  apb.pready:
    value: 1
domain_intent:
  clock_domains:
    - id: pclk_domain
      clock: PCLK
  reset_domains:
    - id: presetn_domain
      reset: PRESETn
      assertion: asynchronous
      deassertion: synchronous
  cdc_crossings:
    - id: CDC_GPIO_I_TO_PCLK
      crossing_type: multi_bit_level_sample
      allowed_pattern: per_bit_two_stage_sync
```

Without this level of detail, an LLM tends to create plausible hardware instead
of contract-correct hardware.

## RTL Trace

RTL files do not need excessive comments, but contract trace can be useful:

```verilog
// OAG_CONTRACT: CONTRACT_APB_DATA_OUT_RW
// OAG_BEHAVIOR_REF: behavior_model.registers.DATA_OUT.write
// OAG_CYCLE_REF: cycle_rules.apb.write_sample
always @(posedge PCLK or negedge PRESETn) begin
  if (!PRESETn) begin
    data_out_q <= '0;
  end else if (apb_write && addr_data_out) begin
    data_out_q <= PWDATA[GPIO_WIDTH-1:0];
  end
end
```

Inline trace comments are useful but not the only acceptable mechanism. At
minimum, the generated trace matrix must connect:

```text
contract -> RTL file/block/comment or line anchor -> TB scenario -> scoreboard row
```

## RTL Handoff Receipt

An RTL child receipt should report implemented contracts and refs. It must not
claim final closure.

```json
{
  "status": "RTL_HANDOFF_PASS",
  "rtl_dialect": "oag_sv_lite_v1",
  "implemented_contracts": [
    "CONTRACT_APB_DATA_OUT_RW",
    "CONTRACT_APB_DIR_RW"
  ],
  "behavior_refs_implemented": [
    "behavior_model.registers.DATA_OUT.write",
    "behavior_model.registers.DATA_OUT.read"
  ],
  "cycle_rule_refs_implemented": [
    "cycle_rules.apb.write_sample",
    "cycle_rules.apb.read_response",
    "cycle_rules.apb.pready_always_1"
  ],
  "changed_paths": [
    "apb_gpio/rtl/apb_gpio.sv",
    "apb_gpio/list/rtl.f"
  ],
  "checks_run": [
    "iverilog_parse",

      "APB response is single-cycle with PREADY tied high by contract.",
      "Address decode is simple; no pipeline added for simple peripheral profile."
    ],
    "power": [
      "Architectural registers update only on mapped writes or events.",
      "GPIO input synchronizer updates every cycle because input sampling is required."
    ],
    "area": [
      "No unnecessary replicated datapath.",
      "No pipeline registers beyond required synchronizer and architectural state."
    ],
    "tradeoffs": [
      "Single-cycle APB decode favors simplicity/area over high-frequency pipelining; acceptable for simple_leaf_apb_peripheral."
    ]
  },
  "domain_crossing_notes": {
    "clock_domains": ["pclk_domain"],
    "reset_domains": ["presetn_domain"],
    "cdc_structures_implemented": [
      {
        "crossing_id": "CDC_GPIO_I_TO_PCLK",
        "pattern": "per_bit_two_stage_sync",
        "rtl_refs": ["gpio_i_sync1_q", "gpio_i_sync2_q"],
        "contract_refs": ["CONTRACT_GPIO_I_SYNC"]
      }
    ],
    "rdc_structures_implemented": [],
    "open_domain_crossing_blockers": []
  },
  "may_claim_complete": false
}
```

`may_claim_complete` must be false for RTL implementation agents. Closure
belongs to evidence validation, gate review, and OAG decision receipts.

## RTL Static Validation

RTL static validation cannot fully prove behavior, but it should check:

- RTL file exists;
- filelist updated;
- module name matches structure;
- ports match structure;
- parameters match structure;
- reset polarity matches cycle rules;
- APB signals are used consistently;
- `PREADY` policy is implemented;
- unmapped access `PSLVERR` policy is implemented when required;
- declared behavior refs have corresponding implementation trace;
- declared cycle refs have corresponding implementation trace;
- RTL dialect policy is respected;
- PPA notes are present for nontrivial RTL;
- no PPA optimization changes contract behavior;
- domain intent is present when crossings, async inputs, multi-clock, or
  multi-reset logic are in scope;
- CDC/RDC structures map to approved crossing patterns;
- lightweight `oag_domain_crossing_check.py` has run or an explicit rationale
  explains why it is not applicable;
- no ontology/spec file was modified by the RTL child;
- compile/lint parse passes;
- lightweight `oag_ppa_check.py` has run or an explicit rationale explains why
  it is not applicable;
- receipt path and dispatch path match.

TB/sim validation then checks the stronger semantic claims:

- scoreboard expected source is not RTL-derived;
- observed source is DUT-facing;
- scenario maps to the same contract RTL claims to implement;
- validation record exists;
- gate reviewed fresh evidence.

## RTL Language Subset

Default generated RTL should target OAG SV-lite:

```text
Verilog-2001 baseline
plus logic
plus generate/genvar/generate-for for static elaboration
```

Allowed by default:

- `module`, `parameter`, `localparam`;
- `input`, `output`, `inout`;
- `wire`, `reg`, `logic`;
- `always @(*)` and edge-triggered `always @(posedge/negedge ...)`;
- blocking/nonblocking assignments used consistently;
- `case`, `if`, ternary expressions;
- `generate`, `genvar`, `for` inside generate.

Not allowed by default:

- `always_ff` and `always_comb`;
- procedural `for`, `while`, `repeat`, or `forever` loops outside generate;
- classes, randomization, interfaces, packages, DPI, assertions inside RTL
  unless the IP profile and toolchain explicitly approve them;
- behavioral delays in synthesizable RTL;
- unsized or ambiguous width behavior where a contract width is known.

The language subset is a portability policy, not a model-depth policy. It does
not weaken behavior/cycle obligations.
